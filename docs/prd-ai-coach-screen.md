# PRD: Экран AI Financial Coach (MVP)

## 1) Product Scope
- **Цель:** отдельный экран, где AI анализирует траты по подпискам и даёт короткие, понятные советы для экономии.
- **Формат:** только **карточки-инсайты** (без чата), **read-only** (без автодействий).
- **AI-провайдер:** OpenRouter (модель выбирается на бэкенде, UI абстрагирован от конкретной модели).

## 2) Ключевая ценность
Пользователь за 10–20 секунд понимает:
1. Где теряет деньги.
2. На чём можно сэкономить в ближайший месяц.
3. Какой совет самый приоритетный.

## 3) Целевая аудитория
- Люди с 3+ активными подписками, которые хотят снизить ежемесячные расходы без сложной аналитики.
- Пользователи, которым важны быстрые выводы, а не глубокая финансовая экспертиза.

## 5) UX-концепт экрана
- **Header:** «AI Coach» + подзаголовок «Советы на основе ваших подписок».
- **Summary block:** 1 главный инсайт (например: «Потенциал экономии до $18/мес»).
- **Insights list:** 3–5 карточек, каждая содержит:
  - Заголовок совета (1 строка)
  - Короткое объяснение «почему»
  - Оценка эффекта (например, «до $4/мес»)
  - Confidence tag: «Высокая / Средняя»
- **Footer note:** «Советы информационные, финальное решение за вами.»

## 6) Состояния интерфейса (обязательно в дизайне)
- **Loading:** skeleton карточек + текст «Анализируем траты…»
- **Success:** отображение карточек
- **Empty data:** «Недостаточно данных. Добавьте больше подписок/платежей.»
- **Error (AI/API):** «Не удалось получить советы. Попробовать снова.»
- **Stale data:** плашка «Советы обновлены X часов назад»

## 7) Контент и тон
- Тон: дружелюбный, деловой, без осуждения.
- Каждая карточка — максимум 2–3 короткие строки.
- Запрещены категоричные формулировки («обязательно», «гарантированно»); использовать вероятностные («возможно», «скорее всего»).

## 8) Логика рекомендаций (MVP)
- Входные данные: активные подписки, цена, период списания, категория, история платежей.
- Типы советов:
  - Дублирующиеся сервисы
  - Редко используемые дорогие подписки
  - Подписки с ближайшим списанием
  - Рост monthly spend vs прошлый период
- Выход: ранжированный список 3–5 советов по ожидаемой экономии и уверенности.

## 9) Ограничения MVP (Non-Goals)
- Нет чата, нет голоса, нет сценариев «сделать за меня».
- Нет кнопок автодействий (cancel/pause/switch).
- Нет долгосрочного финансового планирования за пределами подписок.

## 10) Метрики успеха
- **Primary:** доля пользователей, открывших экран и просмотревших ≥2 карточек.
- **Secondary:** возврат на экран в течение 7 дней; CTR на связанные экраны подписок (если есть навигационные ссылки).
- **Quality:** доля успешных AI-ответов, средняя latency генерации, пользовательская оценка полезности.

## 11) Требования к интеграции OpenRouter
- Серверный вызов (ключ не в клиенте).
- Таймаут и graceful fallback при ошибках.
- Логирование: модель, latency, токены, тип ошибки.
- Версионирование промпта для A/B итераций качества.

## 12) Что передать дизайнеру (deliverables)
- Wireframe + hi-fi для 5 состояний: loading/success/empty/error/stale.
- Компонент карточки с 2 вариантами плотности контента (короткий/средний текст).
- Спецификация адаптива (малые экраны), accessibility (контраст, dynamic type), и текстовые лимиты.

## 14) Design Tokens for AI Coach Screen

### Typography
- Font family: **Inter**
- `title-xl`: 28 / 700 (заголовок экрана)
- `title-lg`: 22 / 700 (главный инсайт)
- `title-md`: 18 / 600 (секционные заголовки)
- `body-md`: 16 / 400–500 (основной текст карточек)
- `body-sm`: 14 / 400–500 (объяснения)
- `caption`: 12 / 400–500 (метки confidence, timestamp)
- `micro`: 11 / 500 (служебные бейджи)

### Spacing scale
- Базовая сетка: `4, 8, 12, 16, 20, 24, 32`
- Экранные горизонтальные поля: `20`
- Вертикальные интервалы между крупными секциями: `16–24`
- Внутренний padding карточки: `16`

### Radius
- `chip`: `20`
- `input/small card`: `12`
- `content card`: `14`
- `hero/summary`: `20`
- `round button`: `22` (44x44)

### Borders
- Толщина: `1`
- Default border color: `border`
- Focus/active border: `primary`

### State opacities (принятый паттерн)
- Pressed: `opacity 0.8–0.85`
- Selected bg tint: `primary + 8–20% alpha`
- Tag tint (пример): `rgba(13,148,136,0.15)`

## 15) Применение токенов к AI-экрану
- **Экран/фон:** `background`
- **Summary card:** градиент `primary → primaryDark`, текст `white`
- **Insight card:** фон `card`, радиус `14`, padding `16`, border `border`
- **Приоритет/риски в карточках:**
  - Low impact — `info`
  - Medium impact — `warning`
  - High saving potential — `success`
  - Risk/possible overpay — `danger`
- **Текст карточки:**
  - Заголовок: `text`
  - Объяснение: `textSecondary`
  - Метки/дата обновления: `textMuted`
- **Empty/Error blocks:** контейнер `card`, иконки `textMuted`, action retry (если есть) в `primary`

## 16) Ограничения для дизайна
- Не вводить новые цвета вне токенов выше.
- Не использовать новые семейства шрифтов (только Inter).
- Не добавлять новые тени как обязательный стиль; опираться на контраст surface/layer.
- Сохранять dark-first визуальный язык, как на текущих tab-экранах.

## 17) Список экранов, которые дизайнер должен сделать

Дизайнер готовит **ровно 5 экранов AI Coach** (без дополнительных вариантов меню):
1. `AI Coach — Loading`
2. `AI Coach — Success`
3. `AI Coach — Empty Data`
4. `AI Coach — Error`
5. `AI Coach — Stale Data`

Важно:
- Это **одно и то же место в приложении** (один раздел AI Coach), а не разные разделы нижнего меню.
- Разница между экранами только в состоянии контента, а не в навигации.

## 18) Нижнее меню (Bottom Tab Bar) — фиксированная спецификация

Для всех 5 экранов из секции 17 используется **одинаковое** нижнее меню.

### Порядок пунктов (слева направо)
1. **Dashboard** — иконка `house`
2. **Subscriptions** — иконка `list`
3. **AI Coach** — иконка `sparkles` (**active** на всех AI-экранах)
4. **Analytics** — иконка `pie-chart`
5. **Reminders** — иконка `notifications`

### Визуальные правила tab bar
- Активный пункт: цвет `primary` (`#0D9488`)
- Неактивные пункты: цвет `textMuted` (`#64748B`)
- Фон tab bar: `background` (`#0F172A`)
- Верхняя граница tab bar: `border` (`#334155`), толщина `1`
- Подписи пунктов — **строго как указано выше**, без переименований (`Home/Insights/Budget/Profile` не использовать)

### Правило консистентности
- На `Loading`, `Success`, `Empty`, `Error`, `Stale` меню не меняется: те же 5 пунктов, те же подписи, те же иконки, активен `AI Coach`.
